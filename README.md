# Merveilles

ブラウザベースのミニMMORPG。グリッド型のダンジョン探索、モンスター戦闘、プレイヤー間協力をリアルタイムで提供する。
DEEO によるコーディング、Aliceffekt (XXIIVV) によるデザイン。

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| サーバー | PHP + MySQL (セッション認証) |
| クライアント | HTML5 / CSS3 / JavaScript (MooTools 1.2.4 + jQuery 1.8.1) |
| 描画 | 16x16 スプライトシート (GIF) + CSS background-position |
| 通信 | 4秒間隔の AJAX ポーリング (JSON) |
| 音響 | フロア連動の MP3 BGM (6トラック) |

## アーキテクチャ

```
Client (merveilles.js)          Server (map.php + merveilles.php)
  |                                |
  |-- GET map.php?action=...  -->  |-- アクション処理 (攻撃/回復/移動/チャット)
  |                                |-- DB更新 (プレイヤー/モンスター/特殊タイル)
  |<-- JSON (状態, マップ, 他PL) --|
  |-- refresh() で再描画           |
  |-- 4秒後に再ポーリング          |
```

サーバーはステートレス設計。各リクエストにプレイヤー座標を含め、サーバー側で全アクションを処理してJSON応答を返す。WebSocketは使わず、AJAXポーリングだけでリアルタイムマルチプレイヤーを実現している。

## データベース設計

3テーブル構成:

### xiv_merveilles (プレイヤー)

| カラム | 用途 |
|--------|------|
| mv_name / mv_password | 認証 (3文字、英数字のみ) |
| x, y, floor | 現在位置 |
| max_floor | 到達最深階 |
| xp | 経験値 (レベル = floor(cbrt(xp))) |
| hp, mp | 体力・魔力 (上限30) |
| kill / save | 撃破数 / セーブ回数 (ビルド値算出用) |
| message | チャットメッセージ |
| avatar_head / avatar_body | 外見 (登録時ランダム) |
| warp1-4 | ワープ魔法のアンロック状態 |
| mv_time | ハートビート (オンライン判定用) |

### xiv_merveilles_monsters (モンスター)

フロアごとに動的に生成・管理。位置 (x, y, floor)、HP、タイムスタンプを保持。

### xiv_merveilles_special (特殊タイル)

ポータルや階段の転送先情報を格納。

## ゲームシステム

### マップ生成

- 1フロア = 64x64 タイルグリッド
- シード値 `(floor * 4) + section` による疑似ランダム生成
- 壁 5%、モンスター 2% の確率で配置
- 初回アクセス時に生成し `/levels/` に PHP serialize() 形式でキャッシュ

### タイルタイプ

| 値 | 意味 |
|----|------|
| 0 | 空地 (移動可) |
| 1 | 壁 |
| 2 | モンスター |
| 3 | 倒されたモンスター |
| 4 / 5 | 上り階段 / 下り階段 |
| 6-11 | 不可視壁・不可視階段・水 |
| 20+ | 壁バリアント |

### 戦闘

- 隣接モンスターをクリックで攻撃
- モンスターHP = `floor(フロア + ((x+y)/64) * (フロア+1))`
- プレイヤー攻撃力 = `floor(レベル * 1.8) - floor(フロア / 3)`
- 被ダメージ = `floor(フロア / 2)`
- 撃破時にXPを獲得、レベルアップで攻撃力が上昇

### マルチプレイヤー

- 同一フロアのプレイヤーをリアルタイム表示
- 隣接プレイヤーへの回復魔法 (MP消費)
- 5秒間表示されるチャットメッセージ
- `mv_time` による10秒以内のオンライン判定

### レベリング

- レベル = `floor(cbrt(xp))` (経験値の立方根)
- ビルド値 = `round((kill - save + 26) / 4)` (0-13、外見に反映)

## 実装思想

1. **シンプルさ優先**: WebSocket不使用、AJAXポーリングのみでリアルタイム性を確保。帯域最小化のためJSONペイロードは軽量に保つ
2. **クロスプラットフォーム**: iPod/iPad/Android/webOS/デスクトップ対応。プラグイン不要のHTML5ベース
3. **協力的マルチプレイ**: 回復魔法をプレイヤー間インタラクションの中心に据え、PvPではなく協力を促進
4. **手続き的生成 + 永続化**: シード値による再現可能なマップ生成と、ファイルキャッシュによる高速ロードの両立
5. **ピクセルアート美学**: カスタムフォント "Marvelous3x3"、統一されたスプライトシート、フロア連動BGMによる一貫した世界観
6. **段階的難易度**: フロアが深くなるほどモンスターのHP・攻撃力が上昇し、レベルとの差分で獲得XPを調整
